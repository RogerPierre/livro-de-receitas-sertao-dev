# 🌵 Livro de Receitas – Sertão Dev

Bem-vindo ao **Livro de Receitas – Sertão Dev**, um projeto colaborativo desenvolvido para a disciplina de **Engenharia de Software I**.  
Este repositório reúne receitas típicas do sertão, utilizando práticas reais de **controle de versão, colaboração, padronização e documentação com Git e GitHub**.

---

## 🎯 Objetivo do Projeto

Aplicar conceitos da Gerência de Configuração de Software na prática:

✔ Uso de **Git e GitHub** com controle de versão  
✔ Branches bem definidas (main, develop, feature/)  
✔ Contribuição via **Pull Requests**  
✔ Uso de **issues**, **commits padronizados** e revisões  
✔ Documentação organizada e navegável  

---

## 🧪 Tecnologias Utilizadas

- Git & GitHub  
- Markdown (.md)  
- Versionamento colaborativo  
- Pull Requests e Code Review

## 🧾 Regras Gerais

- Nenhum commit direto na `main`;  
- Commits pequenos e bem descritos;  
- Cada receita em uma branch separada;  
- Todo PR precisa de **3 aprovações**;  
- Testar links e formatação no GitHub antes de enviar o PR.
