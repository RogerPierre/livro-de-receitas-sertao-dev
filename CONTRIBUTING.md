# 🤝 Guia de Contribuição – Sertão Dev

Obrigado por querer contribuir com o **Livro de Receitas – Sertão Dev**! Este guia explica como colaborar seguindo as práticas adotadas no projeto.

---

## 📌 Princípios Gerais

* **Não faça commits diretamente na branch ****main**.
* Cada alteração deve ser enviada em uma **branch separada**.
* **Commits pequenos**, objetivos e usando mensagens claras.
* Todo Pull Request (PR) deve ser revisado e **aprovado por 3 integrantes**.
* Verifique sempre se a **formatação Markdown** e os **links** estão funcionando corretamente no GitHub.

---

## 🌱 Fluxo de Trabalho (Workflow)

### 1. **Crie uma Issue**

* Antes de começar qualquer alteração, abra uma Issue descrevendo:

  * Tipo de mudança (nova receita, correção, reorganização, etc.)
  * Objetivo
  * Arquivos afetados

### 2. **Crie uma Branch**

Use o padrão:

```
feature/nome-da-receita
fix/ajuste-descricao
refactor/organizar-pastas
```

### 3. **Faça Commits Padronizados**

Siga este modelo:

```
<tipo>: <mensagem curta>
```

Exemplos de tipos:

* `feat` – nova receita ou funcionalidade
* `fix` – correções
* `docs` – documentação
* `refactor` – reorganização de arquivos

Exemplo real:

```
feat: adicionar receita de carne de sol acebolada
```

### 4. **Abra um Pull Request (PR)**

Seu PR deve conter:

* Descrição clara do que foi alterado
* Link para a Issue correspondente

### 5. **Aguarde as Revisões**

* O PR só pode ser mergeado após **3 aprovações**.
* Caso precise ajustar algo, faça novos commits na mesma branch.

## 📂 Onde Colocar sua Receita

Cada categoria possui sua pasta:

* `carnes/`
* `saladas/`
* `sopas/`
* `sobremesas/`
* `massas/`

Nomeie o arquivo em Markdown seguindo o padrão:

```
nome-da-receita.md
```

---

## 🧪 Teste Antes de Enviar

Antes de abrir o PR, confira:

* Formatação do Markdown
* Títulos e subtítulos
* Links para o Menu (`menu.md`) ou outras receitas
* Ortografia e nomes de arquivos
